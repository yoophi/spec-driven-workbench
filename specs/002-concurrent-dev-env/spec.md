# Feature Specification: 중복 실행 가능한 개발환경

**Feature Branch**: `002-concurrent-dev-env`  
**Created**: 2026-06-17  
**Status**: Draft  
**Input**: User description: "프로젝트의 개발환경은 중복으로 실행 가능해야 한다"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 여러 작업 세션을 동시에 실행한다 (Priority: P1)

개발자는 같은 프로젝트를 여러 작업 디렉터리나 여러 terminal 세션에서 동시에 실행해도
각 개발환경이 서로의 실행을 막지 않고 독립적으로 사용할 수 있어야 한다.

**Why this priority**: 중복 실행이 막히면 branch 비교, 병렬 feature 작업, 검토용
실행을 동시에 할 수 없어 개발 흐름이 중단된다.

**Independent Test**: 같은 machine에서 두 개의 독립 작업 디렉터리를 준비한 뒤 각각
개발환경을 시작하고, 두 환경 모두 접근 가능한 상태임을 확인하면 독립적으로 검증할 수
있다.

**Acceptance Scenarios**:

1. **Given** 하나의 개발환경이 이미 실행 중일 때, **When** 개발자가 다른 작업
   디렉터리에서 개발환경을 시작하면, **Then** 두 번째 개발환경도 충돌 없이 시작되고
   접근 가능한 위치를 개발자에게 알려준다.
2. **Given** 두 개 이상의 개발환경이 동시에 실행 중일 때, **When** 개발자가 각
   환경의 web 화면과 backend 상태를 확인하면, **Then** 각 환경은 자신이 속한 작업
   디렉터리의 변경사항과 상태만 보여준다.

---

### User Story 2 - 충돌 상황을 이해하고 복구한다 (Priority: P2)

개발자는 실행에 필요한 local resource가 이미 사용 중이거나 사용할 수 없을 때 원인을
이해하고, 프로젝트 파일을 임의로 수정하지 않고도 다시 실행할 수 있어야 한다.

**Why this priority**: 충돌이 발생했을 때 조용히 실패하거나 모호한 오류만 보이면
개발자가 원인 파악에 시간을 쓰고 임시 수정으로 환경을 오염시킬 수 있다.

**Independent Test**: 실행 충돌을 의도적으로 만든 뒤 개발환경을 시작하고, 오류
메시지와 복구 안내가 개발자가 다음 action을 선택할 수 있을 만큼 명확한지 확인한다.

**Acceptance Scenarios**:

1. **Given** 개발환경 시작에 필요한 local resource가 이미 사용 중일 때, **When**
   개발자가 개발환경을 시작하면, **Then** 시스템은 충돌 원인과 사용 가능한 복구
   경로를 명확히 표시한다.
2. **Given** 개발자가 안내된 복구 경로를 수행했을 때, **When** 개발환경을 다시
   시작하면, **Then** 환경은 정상 시작되거나 남은 문제를 구체적으로 알려준다.

---

### User Story 3 - 동시에 실행 중인 환경을 구분한다 (Priority: P3)

개발자는 여러 개발환경이 동시에 떠 있을 때 어떤 화면과 backend 상태가 어느 작업
디렉터리에 연결되어 있는지 혼동하지 않아야 한다.

**Why this priority**: 중복 실행 자체가 가능해도 환경 구분이 어려우면 잘못된 branch를
검토하거나 변경사항을 오해할 수 있다.

**Independent Test**: 두 환경을 동시에 실행한 뒤 각 환경에서 표시되는 식별 정보와
접근 위치를 비교해 어느 작업 디렉터리의 환경인지 구분 가능하면 검증할 수 있다.

**Acceptance Scenarios**:

1. **Given** 여러 개발환경이 동시에 실행 중일 때, **When** 개발자가 각 환경의 시작
   로그나 상태 표시를 확인하면, **Then** 각 환경을 구분할 수 있는 정보가 표시된다.

### Edge Cases

- 이미 실행 중인 개발환경이 정상 종료되지 않아 local resource가 남아 있는 경우에도
  새 실행은 실패 원인과 복구 방법을 알려야 한다.
- 두 개 이상의 환경을 빠르게 연속으로 시작해도 각 환경의 접근 위치가 서로 섞이거나
  덮어쓰이지 않아야 한다.
- 일부 구성요소만 시작에 성공하고 다른 구성요소가 실패한 경우, 개발자는 어떤
  구성요소가 사용 가능한지와 어떤 구성요소를 복구해야 하는지 확인할 수 있어야 한다.
- 중복 실행을 지원하기 위해 개인 프로젝트 범위를 넘어서는 외부 orchestration이나
  traffic-scale infrastructure를 요구해서는 안 된다.
- 오류와 복구 안내는 한국어 prose로 작성하되, 명령, 경로, identifier는 정확성을 위해
  원문을 유지할 수 있다.

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 개발자는 같은 machine에서 최소 두 개의 독립 작업 디렉터리 또는 실행
  세션으로 개발환경을 동시에 시작할 수 있어야 한다.
- **FR-002**: 동시에 시작된 각 개발환경은 서로 다른 접근 위치를 제공하고, 시작 완료
  시 해당 위치를 개발자에게 명확히 표시해야 한다.
- **FR-003**: 동시에 실행 중인 각 개발환경은 자신이 시작된 작업 디렉터리의 상태와
  변경사항을 기준으로 동작해야 하며, 다른 실행 세션의 상태를 덮어쓰거나 공유해서는
  안 된다.
- **FR-004**: 필요한 local resource가 이미 사용 중인 경우 시스템은 조용히 실패하지
  않고 충돌 원인, 영향을 받은 구성요소, 가능한 복구 action을 표시해야 한다.
- **FR-005**: 개발자는 중복 실행을 위해 repository에 추적되는 설정 파일을 수동으로
  수정하지 않아도 되어야 한다.
- **FR-006**: 개발환경 시작 과정은 pending, success, failure 상태를 개발자가 구분할
  수 있게 표시해야 한다.
- **FR-007**: 부분 실패가 발생한 경우 시스템은 사용 가능한 구성요소와 실패한
  구성요소를 구분해 표시하고, 재시도 또는 종료 후 재실행 경로를 제공해야 한다.
- **FR-008**: 중복 실행 관련 문서와 안내는 한국어 prose로 작성하고, technical term,
  command, path, identifier는 필요한 경우 원문을 유지해야 한다.
- **FR-009**: 이 feature는 local personal workflow 개선에 한정되며, 원격 배포,
  다중 사용자 운영, traffic-scale 최적화를 요구하지 않아야 한다.

### Key Entities

- **개발환경 실행 세션**: 개발자가 특정 작업 디렉터리에서 시작한 local 실행 단위.
  시작 상태, 접근 위치, 연결된 작업 디렉터리, 구성요소별 상태를 가진다.
- **Local resource 충돌**: 개발환경 시작에 필요한 machine-local 자원이 이미 사용
  중이거나 사용할 수 없는 상태. 원인, 영향을 받은 구성요소, 복구 action으로 설명된다.
- **환경 식별 정보**: 개발자가 동시에 실행 중인 환경을 구분하기 위해 확인하는 정보.
  작업 디렉터리, branch 또는 표시 이름, 접근 위치를 포함할 수 있다.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 개발자는 같은 machine에서 두 개의 독립 작업 디렉터리 개발환경을 5분
  이내에 동시에 시작하고 각각 접근할 수 있다.
- **SC-002**: 첫 번째 개발환경이 이미 실행 중인 상태에서 두 번째 개발환경을 시작할 때
  resource 충돌로 인한 시작 실패가 발생하지 않는다.
- **SC-003**: 의도적으로 충돌 상황을 만든 경우, 개발자는 1분 이내에 오류 메시지만으로
  영향을 받은 구성요소와 다음 복구 action을 식별할 수 있다.
- **SC-004**: 동시에 실행 중인 두 환경에서 서로 다른 변경사항을 확인할 때, 개발자는
  각 환경이 어느 작업 디렉터리에 연결되었는지 혼동 없이 구분할 수 있다.

## Assumptions

- 대상 사용자는 이 repository를 local machine에서 개발하는 단일 개발자다.
- 중복 실행은 같은 machine 안의 여러 작업 디렉터리 또는 여러 terminal 세션을
  대상으로 한다.
- 원격 preview 환경, container orchestration, shared development server 운영은 이
  feature의 범위 밖이다.
- 기존 개발환경의 설치와 기본 실행 흐름은 유지하며, 중복 실행을 위해 필요한 안내와
  실행 behavior만 개선한다.
- backend 데이터 저장소나 외부 서비스 상태 공유는 현재 범위에 포함되지 않는다.
