# Repository Instructions

- Keep changes focused on the user's requested task.
- Prefer simple, maintainable implementations over broad refactors.
- Do not revert unrelated user changes.
- Run relevant checks or tests before reporting completion when the project has them.
- Use ASCII in new files unless the existing project convention requires otherwise.
- Follow `.specify/memory/constitution.md` for spec-driven workflow and quality gates.
- Write spec-related documents in Korean; keep technical terms and code exact when needed.
- For frontend code, follow Feature-Sliced Design and build UI with Tailwind CSS and shadcn/ui.
- For backend code, use Hexagonal Architecture and include unit and integration tests.
